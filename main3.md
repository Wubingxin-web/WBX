# 校园空间调度智能助手
基于 AI Agent 多智能体架构开发，专注解决校园场地预约管理痛点，实现场地查询、智能预约、冲突检测、规则问答全流程自动化，适配校园师生使用场景，轻量易部署、可直接运行。

## 📋 项目概述
本项目是一套面向校园场景的 AI 智能应用，核心基于多Agent协作模式，结合简单RAG知识库检索能力，解决传统校园场地预约流程繁琐、信息不透明、人工调度效率低、预约冲突频发、规则查询不便等核心问题。
项目无需复杂依赖，Python环境即可直接运行，支持自然语言交互，适配学术报告厅、计算机实验室、体育馆3类常见校园场地，实现从用户需求识别到任务执行的全闭环，大幅提升校园场地管理效率与师生使用体验。

## 🎯 核心功能
- **意图智能识别**：通过 Intent Agent 自动解析用户自然语言请求，精准区分「场地查询」「场地预约」「规则咨询」三大核心需求，无需固定指令格式。
- **场地状态查询**：实时获取各类场地的容纳人数、空闲时段，快速反馈场地可用信息，避免盲目预约。
- **智能预约与冲突检测**：由 Scheduler Agent 执行调度决策，自动检查预约时段冲突，冲突时提示替代方案，预约成功后锁定对应时段。
- **规则智能问答**：基于 RAG 知识库 Agent，检索校园场地使用规则，精准反馈对应场地的使用规范，无需人工翻阅文档。
- **轻量易运行**：无复杂第三方依赖，无需配置数据库、API密钥，本地Python环境即可启动，适合快速部署与演示。

## 🧩 系统架构（多Agent协作）
项目采用多Agent分工协作架构，各模块独立运行、协同工作，形成完整的业务闭环，结构清晰、易于扩展：
1. **Intent Agent（意图解析智能体）**：核心交互入口，负责解析用户自然语言请求，提取场地名称、预约时段等关键信息，判断用户核心意图（查询/预约/规则咨询）。
2. **Scheduler Agent（调度决策智能体）**：核心控制中枢，调用工具函数完成场地查询、冲突检测、预约提交等操作，执行长链推理，确保预约流程合规、无冲突。
3. **RAG Agent（知识库智能体）**：加载校园场地使用规则文档，实现规则的精准检索与匹配，当用户咨询规则时，快速返回对应场地的规范要求。
4. **工具函数模块**：提供场地数据初始化、场地查询、冲突检测、预约提交等基础工具，为Agent提供执行支撑，可灵活扩展新功能。

## 🛠️ 技术栈
- 编程语言：Python 3.8+
- 核心架构：多Agent协作模式
- 知识库：本地文档检索（RAG简化实现）
- 依赖：无复杂第三方依赖，原生Python即可运行

## 📁 项目结构
```
WBX/
├── main.py              # 项目主入口，负责交互与Agent调度
├── agents.py            # 多Agent核心逻辑（意图解析、调度决策、知识库）
├── tools.py             # 工具函数（场地数据初始化、查询、预约、冲突检测）
├── rule.txt             # 校园场地使用规则（RAG知识库数据源）
└── README.md            # 项目说明文档（本文档）
```

## 🚀 快速运行指南
### 1. 环境准备
确保本地安装 Python 3.8 及以上版本（无需额外安装依赖）。

### 2. 克隆仓库
```bash
# 克隆本仓库到本地
git clone https://github.com/Wubingxin-web/WBX.git
# 进入项目目录
cd WBX
```

### 3. 启动项目
```bash
python main.py
```

### 4. 使用示例
```
# 启动后输入以下指令即可使用
1. 查询类：查询学术报告厅空闲时间
2. 预约类：预约计算机实验室 10:00-12:00
3. 规则类：查询体育馆使用规则
4. 帮助：输入 help 查看所有指令
5. 退出：输入 exit 关闭系统
```

## 🔧 扩展方向
- 新增场地类型（如会议室、图书馆自习室），只需在 tools.py 中扩展场地数据即可。
- 优化意图识别逻辑，支持更模糊的自然语言表述（如“明天能约报告厅吗”）。
- 增加用户角色管理（学生/教师/管理员），区分不同角色的预约权限。
- 集成本地数据库（如SQLite），实现预约记录持久化，避免程序重启后数据丢失。
- 添加邮件/短信通知功能，预约成功后自动推送提醒。

## 💡 注意事项
- 运行前请确保所有文件在同一目录下，避免路径错误。
- 预约时段需严格按照「HH:MM-HH:MM」格式输入（如10:00-12:00），否则无法识别。
- 程序运行期间，预约数据存储在内存中，重启程序后预约记录会重置（可通过扩展数据库解决）。
- 可修改 rule.txt 文件，自定义校园场地使用规则，RAG Agent 会自动加载新规则。
